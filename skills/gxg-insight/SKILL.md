# 商业洞察 · 从招股书学商业

> GXG 子技能 | 观星哥

## 核心定位

**不是"招股书摘要器"。是"商业逻辑训练工具"。**

招股书是最好的商业教材——里面藏着真实的商业模式、竞争格局、财务结构、行业逻辑。招股书会写"提供智能化整体解决方案"，本 Skill 会告诉你：这是一家卖铲子的设备公司，客户是电池厂，靠卖设备和后续维护赚钱，最大的风险是下游客户扩产放缓。

**目标**：每一次分析，都是在训练你的商业思维——下次看到任何一家公司，你都能快速看穿它的生意本质。

---

## 语言规则

输出语言与用户输入语言一致。

---

## 设计哲学

### 不做
- 不建知识图谱
- 不积累、不记忆
- 不让推理模型读原文（只读压缩后的 Master Fact Sheet）
- 不把推理和写作混在一个 Prompt 里

### 只做
PDF → 事实抽取 → 事实合并 → 商业模式推理 → 报告生成 → 输出 MD

**一次性分析流水线。任务结束，中间数据丢弃。**

---

## 三层架构

| 层 | 工作 | 智力 | 处理方式 |
|---|------|------|---------|
| Layer 1 | 分块事实抽取 | 低 | 逐章处理，结构化 JSON |
| Layer 2 | 事实合并去重 | 中 | 合并去重归类 |
| Layer 3 | 推理 + 报告 | 高 | 只读 3000-10000 Token 的 Master Fact Sheet |

**核心原则：大模型不读原文，只读压缩后的 Fact Sheet。**

---

## 工作流程

### Phase 0：接收输入

**触发**：用户从 GXG 主路由进入，或直接提供招股书文件路径。

1. 确认文件存在且为 PDF（也支持 .txt/.md）
2. 提取文本：`pdftotext -layout "[文件路径]" "[临时目录]/raw.txt"`
3. 估算 Token 数
4. 询问用户选择报告深度（若未指定）：

```
📄 已加载：[文件名]
📊 规模：约 [X] 字

请选择报告深度：
```
- Level 1（30秒看懂·约500字）：一句话+产品+赚钱逻辑+大白话
- Level 2（3分钟看懂·约2500字）：完整模块，每个判断有依据（推荐）
- Level 3（30分钟研究版·约8000字）：充分展开，含财务趋势、对手对比

---

### Phase 1：章节识别与分块

按章节切分。核心章节：风险因素、行业概览、业务、财务资料、募集资金运用、未来计划。

跳过：释义、公司治理、法律意见。

---

### Phase 2：分块事实抽取（Layer 1）

逐章处理。读取 `prompts/extract-facts.md`，对每个核心章节抽取结构化事实 → 输出 JSON。

- 只抽取，不总结，不解释
- 客户名、供应商名完整保留
- 数字精确保留

---

### Phase 3：事实合并（Layer 2）

读取 `prompts/merge-facts.md`，合并所有 `facts_*.json` → `master-fact-sheet.json`。

合并后预期大小：3000-10000 Token（压缩比约 50:1 到 100:1）。

---

### Phase 4：商业模式推理（Layer 3 - Call 1）

**只读 Master Fact Sheet，不读原文。**

读取 `prompts/reason-business.md`，完成：
1. 商业模式分类
2. 客户痛点分析
3. 价值创造分析
4. 赚钱逻辑分析
5. 竞争位置分析
6. 风险与机会判断
7. 匿名实体识别
8. 行业视角与定语剥离

**匿名实体识别要点**：利用模型训练数据知识，将脱敏描述匹配到已知公司。不确定时使用 `webfetch` 交叉验证。`confidence: low` 好过强行匹配。

---

### Phase 5：报告生成（Layer 3 - Call 2）

读取 `prompts/generate-report.md` 和 `templates/report-template.md`。

根据用户选择的深度生成 Markdown 报告。报告包含：
1. 一句话讲清公司
2. 产品和服务
3. 客户是谁
4. 为什么客户付钱
5. 产业链位置
6. 赚钱逻辑
7. 竞争格局
8. 核心风险
9. 核心机会
10. 大白话解释
11. 行业视角（定语剥离）
12. 信息来源

**报告末尾强制统一页脚**：

```
---
*本报告由 GXG · 观星哥财务职场工具箱生成 | 子技能：商业洞察 | 深度：Level X*
*观星哥 | 小红书：观星哥 | 微信：guanxingge2025*
```

保存到招股书所在目录：`[目录]/[公司名称]-[YYYYMMDD]-L[level].md`

---

### Phase 6：清理与交付

1. 清理所有临时 JSON 文件
2. 向用户展示报告摘要

```
✅ 分析完成！

📁 报告已保存：[目录]/宁德时代-20260712-L2.md

📊 报告摘要：
> [一句话讲清公司]
> [产品类型] | [客户类型] | [收入模式]

💡 核心发现：
- [关键发现1]
- [关键发现2]
- [关键发现3]

🧠 商业启发：
- [这次分析让你学到什么商业逻辑]
- [这类商业模式有什么通用特征]
```

## 内置知识

### 商业模式分类

```
设备商：研发生产销售专业设备。特征：B2B、项目制、高客单价、重售后。
软件商：开发销售软件/SaaS。特征：可复制、边际成本低、订阅或许可。
制造商：标准化生产。特征：规模化、成本竞争、产能驱动。
品牌商：自有品牌面向消费者。特征：渠道+品牌双驱动、C端。
渠道商：分销代理。特征：薄利多销、资金周转。
平台商：撮合交易。特征：双边网络效应、轻资产。
服务商：提供专业服务。特征：人效驱动、项目制。
原材料商：基础材料/零部件。特征：大宗商品属性、价格波动。
金融服务商：金融产品和服务。特征：牌照壁垒、资金驱动。
```

### 招股书用语 → 白话翻译

| 招股书用语 | 白话 |
|-----------|------|
| 智能化整体解决方案 | 卖设备/软件+附带服务 |
| 一站式综合服务 | 什么都做，但不一定每样都精 |
| 行业领先 | 可能是真的，也可能只是自封的 |
| 协同效应 | 买来的公司和原有业务有关系 |
| 赋能 | 帮客户做事情 |
| 轻资产运营 | 不自己建工厂 |
| 募投项目 | 用 IPO 融来的钱干的事 |
| 生态化布局 | 上下游都想做 |
| 全栈能力 | 从头到尾都能自己做 |

### 行业排名"定语剥离"

招股书经常通过添加限定词来制造"细分第一"。需要剥离修饰性定语，还原真实行业位置。

**剥离方法**：
1. 找出排名中的所有限定词（地域、时间、产品类型、统计口径）
2. 逐一去掉限定词，问"去掉这个词后排名还成立吗？"
3. 如果去掉后排名不确定或大幅下降，说明这个定语是关键修饰
4. 在"行业视角"章节中指明公司的定语手法，说明去掉定语后的真实竞争格局

## 铁律

1. Phase 2 只抽事实，不总结、不解释
2. Phase 4 不读原文，只读 Master Fact Sheet
3. Phase 4 和 Phase 5 分离执行（推理 ≠ 写作）
4. Phase 6 清理所有中间 JSON
5. 最终报告必须独立可读，不依赖原始招股书
6. 报告末尾必须输出 GXG 统一页脚

---

## 成本参考

以 100 万 Token 招股书为例：
- Phase 2（事实抽取）：~30 万 Token → 60-70% 成本
- Phase 3（合并）：~2 万 Token → 5%
- Phase 4（推理）：~5000 Token → 15%
- Phase 5（写作）：~3000 Token → 10%

相比直接丢全文给大模型，节省约 **60-80%** Token 成本。
